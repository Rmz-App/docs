# عرض اشتراك

<div className="flex items-center gap-2 mb-4">
  <span className="px-2 py-1 bg-green-100 text-green-800 rounded text-sm font-mono font-bold">GET</span>
  <code>/subscriptions/{"{id}"}</code>
</div>

تُعيد هذه النقطة تفاصيل كاملة لاشتراك عميل واحد، بما في ذلك معلومات المنتج والعميل والطلب.

<Snippet file="auth-bearer-header.mdx" />

## الترويسات (Headers)

| الترويسة | القيمة | مطلوبة |
|--------|-------|----------|
| `Authorization` | `Bearer YOUR_API_TOKEN` | نعم |
| `Accept` | `application/json` | مُستحسن |

## معاملات المسار (Path Parameters)

| المعامل | النوع | مطلوب | الوصف |
|-----------|------|----------|-------------|
| `id` | integer | نعم | مُعرِّف الاشتراك |

## مثال على الطلب

<CodeGroup>
```bash cURL
curl -X GET "https://merchant-api.rmz.gg/shawarma/subscriptions/501" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```javascript JavaScript
const response = await fetch("https://merchant-api.rmz.gg/shawarma/subscriptions/501", {
  headers: { "Authorization": "Bearer YOUR_API_TOKEN" }
});
const subscription = await response.json();
```

```python Python
import requests

response = requests.get(
    "https://merchant-api.rmz.gg/shawarma/subscriptions/501",
    headers={"Authorization": "Bearer YOUR_API_TOKEN"}
)
subscription = response.json()
```

```php PHP
$ch = curl_init("https://merchant-api.rmz.gg/shawarma/subscriptions/501");
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Authorization: Bearer YOUR_API_TOKEN"
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$response = json_decode(curl_exec($ch), true);
```
</CodeGroup>

## حقول الاستجابة

| الحقل | النوع | الوصف |
|-------|------|-------------|
| `id` | integer | مُعرِّف الاشتراك |
| `status` | string | الحالة الحالية: `trialing`, `active`, `past_due`, `paused`, `canceled`, `expired` |
| `external_customer_id` | string/null | مُعرِّف ثابت يُوفِّره التاجر عند إنشاء جلسة الدفع. استخدمه لربط اشتراكات RMZ بسجلات المستخدمين في نظامك. يكون `null` إذا لم يتم توفيره. |
| `current_period_start` | string | بداية فترة الفوترة الحالية (بصيغة ISO 8601) |
| `current_period_end` | string | نهاية فترة الفوترة الحالية (بصيغة ISO 8601) |
| `trial_ends_at` | string/null | تاريخ انتهاء الفترة التجريبية (`null` إذا لم توجد فترة تجريبية) |
| `cancel_at_period_end` | boolean | ما إذا كان الاشتراك سيُلغى في نهاية الفترة الحالية |
| `canceled_at` | string/null | وقت طلب الإلغاء |
| `paused_at` | string | طابع زمني بصيغة ISO 8601 يُسجِّل وقت إيقاف الاشتراك مؤقتًا. يظهر فقط عندما تكون `status` بقيمة `paused`. |
| `paused_remaining_days` | integer | عدد الأيام المتبقية في الفترة الحالية لحظةَ الإيقاف المؤقت. عند استئناف الاشتراك، يُضاف هذا العدد من الأيام إلى نهاية الفترة الجديدة. يظهر فقط عندما تكون `status` بقيمة `paused`. |
| `next_retry_at` | string | طابع زمني بصيغة ISO 8601 لموعد المحاولة التالية لاستيفاء الدفع تلقائيًا. يظهر فقط عندما تكون `status` بقيمة `past_due`. |
| `duration` | string | مفتاح المدة: `monthly`, `quarterly`, `semiAnnual`, `annually`, `biennial`, `quinquennial`, `decennial` |
| `order_id` | integer | مُعرِّف الطلب الأصلي |
| `auto_renew` | boolean | ما إذا كان التجديد التلقائي نشطًا (توجد بطاقة محفوظة ولم يُطلب الإلغاء) |
| `price` | object | سعر الاشتراك المُثبَّت (يُحدَّد عند الإنشاء ولا يتأثر بتغييرات أسعار المتغيّرات) |
| `metadata` | object/null | أزواج مفاتيح وقيم مُخصَّصة يتم إرفاقها عند إنشاء جلسة الدفع |
| `product` | object | منتج الاشتراك (`id`, `name`, `slug`, `type`) |
| `variant` | object | متغيّر الاشتراك الحالي (`id`, `duration`, `duration_text`, `price`) |
| `payment_method` | object/null | بيانات البطاقة المحفوظة (`last_four`, `scheme`) — تظهر فقط إذا كان التجديد التلقائي نشطًا |
| `scheduled_variant` | object/null | تغيير متغيّر قيد الانتظار (يُطبَّق عند التجديد التالي) |
| `is_active` | boolean | ما إذا كان الاشتراك يمنح الوصول حاليًا |
| `is_expired` | boolean | ما إذا كان الاشتراك قد انتهى |
| `days_remaining` | integer | عدد الأيام المتبقية في الفترة الحالية |

## استجابات الأخطاء

| الرمز | الوصف |
|------|-------------|
| `401` | غير مُصرَّح — الرمز (token) غير صالح أو مفقود |
| `404` | الاشتراك غير موجود |
