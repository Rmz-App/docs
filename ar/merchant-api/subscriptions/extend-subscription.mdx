# تمديد اشتراك

<div className="flex items-center gap-2 mb-4">
  <span className="px-2 py-1 bg-blue-100 text-blue-800 rounded text-sm font-mono font-bold">POST</span>
  <code>/subscriptions/{"{id}"}/extend</code>
</div>

يُمدِّد هذا المسار الفترة الفوترة الحالية لاشتراك بعدد محدد من الأيام أو الأشهر. يُستخدم عادةً لتمديدات المجاملة، أو للتعويض عن انقطاعات الخدمة، أو لتقديم عروض ترويجية. يجب تمرير إمّا `days` أو `months` (وليس كليهما معًا).

<Warning>
  يحتوي هذا المسار على حماية ضدّ التكرار (idempotency guard). إذا أُرسل طلب مكرّر خلال 10 ثوانٍ، فسيُردّ بـ `429 Too Many Requests` بدلًا من تمديد الاشتراك مرتين. هذه الحماية تمنع تكرار التمديد عن طريق الخطأ عند وصول طلبات متزامنة.
</Warning>

## الترويسات (Headers)

| Header | القيمة | مطلوب |
|--------|-------|----------|
| `Authorization` | `Bearer YOUR_API_TOKEN` | نعم |
| `Content-Type` | `application/json` | نعم |
| `Accept` | `application/json` | مستحسن |

## معاملات المسار (Path Parameters)

| Parameter | النوع | مطلوب | الوصف |
|-----------|------|----------|-------------|
| `id` | integer | نعم | معرّف الاشتراك |

## جسم الطلب (Request Body)

| Parameter | النوع | مطلوب | الوصف |
|-----------|------|----------|-------------|
| `days` | integer | مطلوب في حال عدم تمرير `months` | عدد الأيام المراد تمديد الاشتراك بها (من 1 إلى 365) |
| `months` | integer | مطلوب في حال عدم تمرير `days` | عدد الأشهر المراد تمديد الاشتراك بها (من 1 إلى 24). يُعامل كل شهر على أنه 30 يومًا. |

## مثال على الطلب

<CodeGroup>
```bash cURL
curl -X POST "https://merchant-api.rmz.gg/shawarma/subscriptions/501/extend" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"days": 7}'
```

```javascript JavaScript
const response = await fetch(
  "https://merchant-api.rmz.gg/shawarma/subscriptions/501/extend",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer YOUR_API_TOKEN",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ days: 7 })
  }
);
const result = await response.json();
```

```python Python
import requests

response = requests.post(
    "https://merchant-api.rmz.gg/shawarma/subscriptions/501/extend",
    headers={
        "Authorization": "Bearer YOUR_API_TOKEN",
        "Content-Type": "application/json"
    },
    json={"days": 7}
)
result = response.json()
```

```php PHP
$ch = curl_init("https://merchant-api.rmz.gg/shawarma/subscriptions/501/extend");
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Authorization: Bearer YOUR_API_TOKEN",
    "Content-Type: application/json"
]);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode(["days" => 7]));
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$response = json_decode(curl_exec($ch), true);
```
</CodeGroup>

## استجابة النجاح

يعيد المسار كائن الاشتراك الكامل بالشكل القياسي `SubscriptionResource` — وهو مطابق لما تعيده مسارات [عرض اشتراك](./get-subscription) و[قائمة الاشتراكات](./list-subscriptions) و[البحث عن اشتراكات](./lookup-subscriptions). فيما يلي الحقول الأكثر صلةً بعملية التمديد — راجع [عرض اشتراك](./get-subscription) للاطلاع على القائمة الكاملة للحقول.

```json
{
  "message": "Subscription extended successfully",
  "data": {
    "id": 501,
    "status": "active",
    "external_customer_id": "usr_abc123",
    "current_period_start": "2025-06-01T00:00:00.000000Z",
    "current_period_end": "2025-07-08T00:00:00.000000Z",
    "end_date": "2025-07-08T00:00:00.000000Z"
  },
  "api": "rmz.shawarma",
  "timestamp": 1699999999
}
```

## حقول الاستجابة

الاستجابة هي كائن `SubscriptionResource` الكامل. راجع [عرض اشتراك](./get-subscription) للاطلاع على القائمة الكاملة. الحقول الأكثر صلةً بالتمديد:

| Field | النوع | الوصف |
|-------|------|-------------|
| `id` | integer | معرّف الاشتراك |
| `status` | string | الحالة الحالية للاشتراك |
| `external_customer_id` | string/null | المعرّف الثابت الذي زوّده التاجر عند إنشاء جلسة الدفع. استخدمه لربط اشتراكات RMZ بسجلات المستخدمين لديك. يكون `null` إن لم يُمرَّر. |
| `current_period_start` | string | بداية فترة الفوترة الحالية |
| `current_period_end` | string | نهاية فترة الفوترة الحالية بعد التمديد |
| `end_date` | string | تاريخ انتهاء الاشتراك بعد التحديث |

<Tip>
  يُطلَق حدث الـ webhook `subscription.updated` عند تمديد الاشتراك، ويتضمن عدد الأيام المضافة وتاريخ الانتهاء الجديد.
</Tip>

## استجابات الأخطاء

| Code | الوصف |
|------|-------------|
| `401` | غير مصرّح — الرمز (token) غير صالح أو مفقود |
| `404` | الاشتراك غير موجود |
| `409` | الاشتراك ملغى أو منتهٍ ولا يمكن تمديده |
| `422` | خطأ في التحقق — يجب تمرير `days` (من 1 إلى 365) أو `months` (من 1 إلى 24) |
