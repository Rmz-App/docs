# إنشاء جلسة دفع

<div className="flex items-center gap-2 mb-4">
  <span className="px-2 py-1 bg-blue-100 text-blue-800 rounded text-sm font-mono font-bold">POST</span>
  <code>/subscriptions/checkout-sessions</code>
</div>

يُنشئ هذا المسار جلسة دفع مستضافة تُحوِّل العميل لإتمام عملية الاشتراك. هذه هي الطريقة الأساسية لتكاملات SaaS لإنشاء اشتراكات برمجيًا.

## المصادقة

<Snippet file="auth-bearer-header.mdx" />

## الترويسات (Headers)

| Header | القيمة | مطلوب |
|--------|-------|----------|
| `Authorization` | `Bearer YOUR_API_TOKEN` | نعم |
| `Content-Type` | `application/json` | نعم |
| `Accept` | `application/json` | مستحسن |

## جسم الطلب (Request Body)

| Parameter | النوع | مطلوب | الوصف |
|-----------|------|----------|-------------|
| `product_id` | integer | نعم | معرّف منتج الاشتراك |
| `variant_id` | integer | نعم | معرّف متغيّر الاشتراك (يحدد المدّة والسعر) |
| `customer.id` | integer | مشروط | معرّف عميل موجود. إذا مُرِّر، يُتجاهَل بقية حقول العميل. |
| `customer.country_code` | string | مشروط | مفتاح الاتصال الدولي للعميل (مثال: `"966"` للسعودية، `"971"` للإمارات). مطلوب إذا لم يُمرَّر `customer.id`. |
| `customer.phone` | string | مشروط | رقم جوّال العميل من 5 إلى 15 رقمًا (مثال: `"512345678"`). مطلوب إذا لم يُمرَّر `customer.id`. |
| `customer.firstName` | string | مشروط | الاسم الأول للعميل. مطلوب إذا لم يُمرَّر `customer.id`. |
| `customer.lastName` | string | مشروط | اسم العائلة للعميل. مطلوب إذا لم يُمرَّر `customer.id`. |
| `customer.email` | string | مشروط | البريد الإلكتروني للعميل. مطلوب إذا لم يُمرَّر `customer.id`. |
| `success_url` | string | نعم | الرابط الذي يُحوَّل إليه العميل بعد نجاح الدفع |
| `cancel_url` | string | نعم | الرابط الذي يُحوَّل إليه العميل عند إلغاء العملية |
| `metadata` | object | لا | أزواج مفتاح/قيمة تُربط بالاشتراك. الحد الأقصى 20 مفتاحًا، وكل قيمة حتى 500 حرف. تُضمَّن في جميع حمولات الـ webhook واستجابات الـ API. استخدمها لأي بيانات مخصّصة إضافية تريد إرجاعها مع الاشتراك. |
| `external_customer_id` | string | لا | معرّف المستخدم الثابت في نظامك (حتى 191 حرفًا). معرّف العميل الخارجي هو المفتاح الذي يربط اشتراكات RMZ بمستخدمي نظامك. يُرجَع في كل ردود الـ API وفي جميع الـ webhooks لهذا الاشتراك — في كل حمولة webhook تحت `data.subscription.external_customer_id`، وفي كل استجابة API للاشتراك تحت `data.external_customer_id` (عرض، قائمة، بحث، إلغاء، إيقاف مؤقت، استئناف، تمديد). كما يمكن استخدامه معاملًا في مسار `/subscriptions/lookup`. |

<Info>
  يجب تمرير إمّا `customer.id` (للإشارة إلى عميل موجود) **أو** المجموعة الكاملة من `customer.country_code` و`customer.phone` و`customer.firstName` و`customer.lastName` و`customer.email` (لإنشاء عميل جديد أو مطابقة عميل موجود).
</Info>

## مثال على الطلب (عميل جديد)

<CodeGroup>
```bash cURL
curl -X POST "https://merchant-api.rmz.gg/shawarma/subscriptions/checkout-sessions" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "product_id": 102,
    "variant_id": 15,
    "customer": {
      "country_code": "966",
      "phone": "512345678",
      "firstName": "Ahmed",
      "lastName": "Ali",
      "email": "ahmed@example.com"
    },
    "metadata": {
      "external_user_id": "usr_abc123",
      "plan": "pro"
    },
    "success_url": "https://yourapp.com/subscription/success",
    "cancel_url": "https://yourapp.com/subscription/cancel"
  }'
```

## استجابة النجاح

```json
{
  "message": null,
  "data": {
    "session_id": "cs_abc123def456",
    "checkout_url": "https://billing.rmz.gg/checkout/01KNAE1283PNWZDNE5W2TNRY5J",
    "expires_at": "2025-06-01T01:00:00.000000Z"
  },
  "api": "rmz.shawarma",
  "timestamp": 1699999999
}
```

## حقول الاستجابة

| Field | النوع | الوصف |
|-------|------|-------------|
| `session_id` | string | المعرّف الفريد لجلسة الدفع |
| `checkout_url` | string | الرابط الذي يجب تحويل العميل إليه لإتمام الدفع |
| `expires_at` | string | طابع زمني بصيغة ISO 8601 يوضح وقت انتهاء الجلسة (بعد 7 دقائق) |

## سير العملية

1. يُنشئ خادمك جلسة الدفع عبر هذا المسار
2. تُحوِّل العميل إلى `checkout_url`
3. يُتمّ العميل عملية الدفع على صفحة الدفع المستضافة
4. عند النجاح، يُحوَّل العميل إلى `success_url` الذي حدّدته مع إضافة `?rmz-subscription-id={id}` كمعامل استعلام
5. يُطلَق حدث الـ webhook `subscription.created` إلى نقطة الاستقبال لديك

**مثال على التحويل:**
```
https://yourapp.com/subscription/success?rmz-subscription-id=501
```

<Tip>
  استخدم معامل الاستعلام `rmz-subscription-id` من عملية التحويل للاستعلام فورًا عن الاشتراك. مع ذلك، استمع دائمًا لحدث الـ webhook `subscription.created` للتأكد من نجاح إنشاء الاشتراك، لأن التحويل وحده لا يضمن نجاح عملية الدفع.
</Tip>

## استجابات الأخطاء

| Code | الوصف |
|------|-------------|
| `401` | غير مصرّح — الرمز (token) غير صالح أو مفقود |
| `404` | المنتج أو المتغيّر غير موجود |
| `422` | خطأ في التحقق — معاملات مفقودة أو غير صحيحة |
