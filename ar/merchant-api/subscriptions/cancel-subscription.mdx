# إلغاء الاشتراك

<div className="flex items-center gap-2 mb-4">
  <span className="px-2 py-1 bg-blue-100 text-blue-800 rounded text-sm font-mono font-bold">POST</span>
  <code>/subscriptions/{"{id}"}/cancel</code>
</div>

يقوم بإلغاء اشتراك عميل. يمكنك الإلغاء فورًا (يُسحب الوصول على الفور) أو في نهاية فترة الفوترة الحالية (يحتفظ العميل بالوصول حتى انتهاء الفترة).

## الترويسات (Headers)

| الترويسة | القيمة | مطلوبة |
|--------|-------|----------|
| `Authorization` | `Bearer YOUR_API_TOKEN` | نعم |
| `Content-Type` | `application/json` | نعم |
| `Accept` | `application/json` | مُستحسنة |

## معاملات المسار (Path Parameters)

| المعامل | النوع | مطلوب | الوصف |
|-----------|------|----------|-------------|
| `id` | integer | نعم | معرّف الاشتراك |

## جسم الطلب (Request Body)

| المعامل | النوع | مطلوب | الوصف |
|-----------|------|----------|-------------|
| `effective` | string | لا | `immediate` أو `end_of_period` (الافتراضي: `end_of_period`) |

## مثال طلب

<CodeGroup>
```bash cURL
curl -X POST "https://merchant-api.rmz.gg/shawarma/subscriptions/501/cancel" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"effective": "end_of_period"}'
```

```javascript JavaScript
const response = await fetch(
  "https://merchant-api.rmz.gg/shawarma/subscriptions/501/cancel",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer YOUR_API_TOKEN",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ effective: "end_of_period" })
  }
);
const result = await response.json();
```

```python Python
import requests

response = requests.post(
    "https://merchant-api.rmz.gg/shawarma/subscriptions/501/cancel",
    headers={
        "Authorization": "Bearer YOUR_API_TOKEN",
        "Content-Type": "application/json"
    },
    json={"effective": "end_of_period"}
)
result = response.json()
```

```php PHP
$ch = curl_init("https://merchant-api.rmz.gg/shawarma/subscriptions/501/cancel");
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Authorization: Bearer YOUR_API_TOKEN",
    "Content-Type: application/json"
]);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode(["effective" => "end_of_period"]));
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$response = json_decode(curl_exec($ch), true);
```

## استجابة النجاح

يُرجع كائن الاشتراك الكامل بالشكل القياسي `SubscriptionResource` — مطابق لكلٍّ من [عرض اشتراك](./get-subscription) و[قائمة الاشتراكات](./list-subscriptions) و[البحث عن الاشتراكات](./lookup-subscriptions). فيما يلي أكثر الحقول صلة بعملية الإلغاء — راجع [عرض اشتراك](./get-subscription) للاطلاع على قائمة الحقول الكاملة.

```json
{
  "message": "Subscription canceled successfully",
  "data": {
    "id": 501,
    "status": "active",
    "external_customer_id": "usr_abc123",
    "cancel_at_period_end": true,
    "canceled_at": "2025-06-15T12:00:00.000000Z",
    "current_period_end": "2025-07-01T00:00:00.000000Z"
  },
  "api": "rmz.shawarma",
  "timestamp": 1699999999
}
```

## خيارات السلوك

| السلوك | الوصف |
|----------|-------------|
| `end_of_period` | يبقى الاشتراك نشطًا حتى `current_period_end`. لن تُجرى أي محاولة تجديد. هذا هو الخيار الافتراضي والموصى به. |
| `immediate` | يُلغى الاشتراك ويُسحب الوصول فورًا. تتحول الحالة إلى `canceled`. |

<Note>
  عند استخدام `end_of_period`، تظل حالة الاشتراك `active` ويُضبط `cancel_at_period_end` على `true`. سينتقل الاشتراك إلى حالة `expired` عند انتهاء الفترة.
</Note>

<Warning>
  لا يمكن التراجع عن الإلغاء الفوري. سيحتاج العميل إلى إنشاء اشتراك جديد. فكِّر في استخدام `end_of_period` لمنح العميل القيمة الكاملة لفترة فوترته الحالية.
</Warning>

## استجابات الأخطاء

| الرمز | الوصف |
|------|-------------|
| `401` | غير مُصرّح — رمز الوصول غير صالح أو مفقود |
| `404` | الاشتراك غير موجود |
| `409` | الاشتراك مُلغى أو منتهي بالفعل |
| `422` | خطأ في التحقق — قيمة `effective` غير صالحة، أو الاشتراك مُلغى/منتهٍ بالفعل |
